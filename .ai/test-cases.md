# 功能测试用例

## 测试说明

用于验证本地 AI 的推荐标签、生成摘要、生成待办、时间提取、优先级推断功能，以及 Native 密码加密存储功能。

点击笔记编辑页的「本地AI整理」按钮触发测试。

---

## 测试用例 1：会议场景

**测试点**：同义词映射、标签提取、待办提取、时间提取、优先级

```
标题：项目周会
内容：今天下午开了项目周会，讨论了Q2的开发计划。前端需要在月底前完成用户中心的改版，后端要优化数据库查询性能，目前有慢查询问题。测试那边发现了一个登录页面的bug，需要尽快修复。下周三再开一次会对齐进度。
```

**预期结果**：

| 项目 | 预期值 |
|------|--------|
| 推荐标签 | 会议、项目、开发 |
| 摘要 | 包含"项目周会"、"开发计划"、"慢查询"等核心信息，不重复标题 |
| 待办 | 修复登录页面bug、优化数据库查询、完成用户中心改版 |
| 时间 | 下周三 |
| 优先级 | high（有"尽快"） |

## Native 密码加密测试用例

### 测试用例 C1：新用户注册后非明文存储

**测试点**：注册时调用 NativeCrypto 生成随机盐值和 SHA-256 哈希，Preferences 不保存明文密码。

**步骤**：
1. 注册新账号 `crypto0609a`，密码 `Pw123456`，昵称 `CryptoTester`。
2. 登录后读取该账号在 Preferences 中的存储记录。
3. 检查记录格式和是否包含明文密码。

**预期结果**：

| 项目 | 预期值 |
|------|--------|
| 存储格式 | `64位hex哈希:32位hex盐值` |
| 记录长度 | 97 |
| 是否包含明文密码 | false |

### 测试用例 C2：正确密码登录

**测试点**：登录时使用 `NativeCrypto.verifyPassword()` 验证加盐哈希。

**步骤**：
1. 使用账号 `crypto0609a`。
2. 输入正确密码 `Pw123456`。
3. 点击登录。

**预期结果**：进入 `pages/Index`，首页显示 `你好，CryptoTester`。

### 测试用例 C3：错误密码登录

**测试点**：错误密码不能通过哈希验证。

**步骤**：
1. 使用账号 `crypto0609a`。
2. 输入错误密码 `Wrong123`。
3. 点击登录。

**预期结果**：停留在 `pages/LoginPage`，不进入首页。

### 测试用例 C4：旧版明文密码迁移

**测试点**：旧版明文密码账号首次正确登录后自动重写为 `hash:salt`。

**步骤**：
1. 构造旧版账号记录：账号 key 对应值为明文密码。
2. 使用该明文密码登录。
3. 重新读取 Preferences 记录。

**预期结果**：登录成功，原明文记录被重写为 `64位hex哈希:32位hex盐值`。
